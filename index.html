O que é um modelo?

Um modelo é uma representação simplificada da realidade, criada para entender ou prever algum aspecto do mundo real.

"Todos os modelos estão errados, mas alguns são úteis."
- George Box, estatístico

Exemplos de modelos no cotidiano

Um mapa é um modelo do território

Uma planta baixa é um modelo de uma casa

Tipos de modelos

  • Modelos conceituais: explicam como um sistema funciona
  • Modelos físicos: representações físicas em escala
  • Modelos matemáticos: usam equações para representar relações
  • Modelos estatísticos: usam dados para estabelecer padrões
  • Modelos computacionais: simulam sistemas complexos

O que é um algoritmo?

Um algoritmo é uma sequência de passos bem definidos para resolver um problema ou realizar uma tarefa.

Algoritmos no cotidiano

Uma receita de bolo

Instruções para montar um móvel

Exemplo: Calculando uma média

  1. Somar todos os valores
  2. Dividir pela quantidade de valores

Exemplo: Para as notas [7, 8, 6, 9]

  1. Soma: 7 + 8 + 6 + 9 = 30
  2. Quantidade: 4
  3. Média: 30 ÷ 4 = 7.5

Modelos de Distribuição de Espécies

Tentam responder à pergunta:

"Baseado nas condições ambientais,
este lugar é adequado para esta espécie viver?"

O processo simplificado

  1. Dados de entrada: Pontos de presença/ausência + Variáveis ambientais
  2. Aprendizado: O algoritmo identifica padrões e relações
  3. Previsão: Estimativa de adequabilidade ambiental

Analogia: O Modelo como um Detetive Ambiental

1. Coletando Pistas

O detetive visita locais onde a planta foi encontrada e anota as condições: quanto chove, qual a temperatura, tipo de solo, etc.

2. Estabelecendo Padrões

Após visitar muitos locais, o detetive percebe: "Esta planta aparece em locais onde chove entre 1000-1500mm por ano e a temperatura média é de 18-25°C."

3. Fazendo Previsões

Agora, quando mostram ao detetive um novo local, mesmo sem visitar, ele pode dizer: "Baseado nas condições, há 80% de chance desta planta viver aqui."

4. Lidando com Incertezas

O detetive sabe que suas previsões têm limitações. Talvez existam outros fatores que ele não considerou, como competição com outras espécies.

O que é Aprendizado de Máquina?

É uma técnica na qual, em vez de programarmos explicitamente todas as regras, deixamos que o computador "aprenda" essas regras a partir dos dados.

Como o Computador "Aprende"?

  1. Dados de Treinamento: Exemplos rotulados (locais com/sem a espécie)
  2. Identificação de Padrões: Correlações entre variáveis ambientais e presença
  3. Aplicação: O modelo faz previsões para novos locais

O que é uma Árvore de Decisão?

Um modelo que funciona fazendo uma série de perguntas sobre os dados, seguindo um caminho até chegar a uma conclusão.

Exemplo: Árvore de Decisão para uma Planta

Temperatura > 20°C?
├── SIM: Precipitação > 1200mm?
│   ├── SIM: Alta probabilidade
│   └── NÃO: Baixa probabilidade
└── NÃO: Altitude > 800m?
    ├── SIM: Média probabilidade
    └── NÃO: Baixa probabilidade

O que é Random Forest?

"Random Forest" significa "Floresta Aleatória" - é um algoritmo que cria muitas árvores de decisão diferentes e combina seus resultados.

Por que usar muitas árvores?

Uma única árvore de decisão tem limitações:

  • Pode ser muito específica para os dados de treinamento (overfitting)
  • Pode mudar drasticamente com pequenas alterações nos dados

Como funciona o Random Forest

  1. Muitas Árvores: Cria dezenas ou centenas de árvores diferentes
  2. Aleatoriedade: Cada árvore usa:
    • Um subconjunto aleatório dos dados
    • Um subconjunto aleatório das variáveis em cada divisão
  3. Votação: Cada árvore dá sua 'opinião' sobre se um local é adequado para a espécie viver (sim/não), e a resposta final é determinada pela maioria, como em uma eleição

Analogia: Uma Equipe de Detetives

Random Forest é como ter uma equipe de detetives, em vez de apenas um:

Como trabalha a equipe de detetives

  • Cada detetive (árvore) recebe uma parte diferente das pistas
  • Cada um usa métodos ligeiramente diferentes para investigar
  • Quando precisamos decidir sobre um novo local, fazemos uma votação entre todos os detetives
  • O resultado final é mais confiável do que confiar em apenas um detetive

Vantagens do Random Forest

  • Precisão: Geralmente oferece previsões mais precisas
  • Robustez: Menos suscetível a ruídos nos dados
  • Transparência: Fornece informações sobre quais variáveis são mais importantes
  • Versatilidade: Funciona bem com diferentes tipos de dados

Parâmetros do Random Forest

Os parâmetros são como "botões de ajuste" que controlam como o algoritmo funciona.

n_estimators: Número de Árvores

  • "Quantos detetives queremos na nossa equipe?"
  • Mais árvores = resultado mais estável
  • Mais árvores = mais tempo de processamento
  • Valor típico: 100-500 árvores

max_depth: Profundidade Máxima

  • "Quão detalhado permitimos que cada detetive seja?"
  • Árvores profundas: capturam relações complexas
  • Árvores muito profundas: risco de "decorar" os dados
  • Limitar profundidade ajuda a generalizar

test_size: Tamanho do Conjunto de Teste

  • "Vamos guardar amostras para verificar se nossos detetives estão acertando"
  • Separamos 20-30% dos dados para testar o modelo
  • Importante para avaliar o desempenho real

max_features: Máximo de Características

  • "Quantos fatores ambientais cada detetive considera de cada vez?"
  • Limitar o número de variáveis aumenta a diversidade entre as árvores
  • Opções comuns:
    • 'sqrt': raiz quadrada do número de variáveis
    • 'log2': log base 2 do número de variáveis
