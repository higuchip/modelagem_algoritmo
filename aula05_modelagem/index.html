Preparação dos Dados

Montando o conjunto de dados

Precisamos juntar tudo que preparamos nas aulas anteriores em uma tabela única para o modelo aprender.

Presenças (GBIF) → label = 1 (presente) + variáveis bioclimáticas → TABELA FINAL Pseudo-ausências (buffer) → label = 0 (ausente) X + y + variáveis bioclimáticas

Divisão Treino-Teste

Por que dividir os dados?

Não podemos avaliar o modelo com os mesmos dados que ele usou para aprender. Seria como o professor aplicar a prova com as respostas escritas no quadro!
Dados de Treino
O modelo aprende os padrões climáticos
Dados de Teste
Avaliamos se o modelo realmente aprendeu

Divisão Treino-Teste

Como dividir: 80/20

Todos os dados (100%) ┌──────────────────────────────────────┬──────────┐ │ TREINO (80%) │TESTE(20%)│ │ │ │ │ O modelo aprende com esses dados │ Avalia │ └──────────────────────────────────────┴──────────┘
Regra fundamental: O modelo NUNCA vê os dados de teste durante o treinamento!
No TAIPA: O slider "Proporção de Teste" controla essa divisão (padrão: 20%).

Divisão Treino-Teste

Estratificação

A proporção de presenças e ausências deve ser mantida em ambos os conjuntos:

200 pontos totais (100 presenças + 100 ausências) Treino (80%): 160 pontos → 80 presenças + 80 ausências ✓ Teste (20%): 40 pontos → 20 presenças + 20 ausências ✓
Estratificação garante que o treino e o teste tenham a mesma proporção de classes. Sem isso, o teste poderia ter só ausências — e a avaliação seria inútil.

Random Forest: Hiperparâmetros

Relembrando

Na Aula 1, vimos que Random Forest é uma equipe de detetives — muitas árvores de decisão votando juntas.

Agora vamos entender os "botões de ajuste" (hiperparâmetros) que controlam como o modelo aprende.

Hiperparâmetros são configurações que nós definimos antes do treinamento. O modelo não os aprende sozinho.

Random Forest: Hiperparâmetros

Os 4 principais

HiperparâmetroO que controlaAnalogia
N. de Árvores Quantas árvores de decisão Quantos detetives na equipe
Profundidade Máx. Quantas perguntas cada árvore faz Quantas perguntas cada detetive pode fazer
Amostras Mín. Split Mínimo de dados para dividir um nó Mínimo de evidências para abrir investigação
Amostras Mín. Folha Mínimo de dados no nó final Mínimo de casos para dar veredito

Random Forest: Hiperparâmetros

Profundidade: raso vs. profundo

Árvore rasa (max_depth = 5)
Poucas perguntas, modelo simples
Risco: underfitting
Árvore profunda (max_depth = 50)
Muitas perguntas, modelo complexo
Risco: overfitting
O ideal está no meio-termo: profundidade suficiente para captar padrões reais, mas não tanta que decore ruído.

Validação Cruzada

O problema do split único

Um único split 80/20 pode dar sorte ou azar. E se os 20% de teste forem pontos "fáceis"? Ou "difíceis"?

Analogia: Avaliar um aluno com uma única prova vs. várias provas ao longo do semestre.

  • Uma prova: nota pode não refletir o conhecimento real
  • Várias provas: média das notas é mais confiável
Validação cruzada é como aplicar várias provas — cada uma com perguntas diferentes.

Validação Cruzada

K-Fold Cross-Validation

Dividimos os dados em K partes iguais (folds). Cada parte é usada como teste exatamente uma vez:

Fold 1: [TESTE] [treino] [treino] [treino] [treino] Fold 2: [treino] [TESTE] [treino] [treino] [treino] Fold 3: [treino] [treino] [TESTE] [treino] [treino] Fold 4: [treino] [treino] [treino] [TESTE] [treino] Fold 5: [treino] [treino] [treino] [treino] [TESTE] Resultado final = média das 5 avaliações

Com K=5, o modelo é treinado e avaliado 5 vezes, cada vez com dados de teste diferentes.

Validação Cruzada

Vantagens do K-Fold

  • Todos os dados são usados para treino E para teste
  • Resultado mais robusto (média de K avaliações em vez de 1)
  • Detecta melhor se o modelo está generalizando
  • Mostra a variabilidade do desempenho (desvio padrão entre folds)
K = 5 é o valor mais comum. Significa 5 rodadas de treino/teste com partes diferentes.
No TAIPA: A validação cruzada vem ativada por padrão. O slider "Número de Folds" permite ajustar (padrão: 5).

Validação Cruzada Espacial

O problema da autocorrelação espacial

Autocorrelação espacial: pontos próximos no espaço tendem a ter condições ambientais similares.

Se um ponto de treino está muito perto de um ponto de teste, o modelo pode "colar" da vizinhança em vez de aprender relações climáticas reais.

Ponto de treino (A) ← 5 km → Ponto de teste (B) A e B têm clima quase idêntico! O modelo acerta B sem ter aprendido nada... apenas porque A estava ao lado.

Validação Cruzada Espacial

Analogia: prova com cola vs. prova real

CV padrão em SDM

Como fazer prova sentado ao lado de quem já sabe as respostas.

Nota alta, mas não aprendeu nada!
CV espacial

Como fazer prova em sala separada, sem consulta.

Nota mais realista — mostra o que realmente aprendeu.

Validação Cruzada Espacial

Como funciona na prática

Passo 1: Agrupar pontos por localização geográfica (KMeans) ┌───────────┐ ┌───────────┐ ┌───────────┐ │ Grupo 1 │ │ Grupo 2 │ │ Grupo 3 │ │ (Norte) │ │ (Centro) │ │ (Sul) │ │ * * * │ │ * * * │ │ * * * │ └───────────┘ └───────────┘ └───────────┘ ↕ buffer ↕ buffer Passo 2: Cada grupo vira conjunto de teste Passo 3: Buffer remove pontos de treino próximos à fronteira

Resultado: métricas refletem a capacidade real de prever em áreas novas.

Validação Cruzada Espacial

Por que é ESSENCIAL para SDM

Validação padrão pode dar AUC de 0.95 quando a capacidade real é 0.75!
  • Dados ecológicos sempre têm autocorrelação espacial
  • O objetivo do SDM é prever em novas áreas
  • Métricas infladas levam a decisões erradas de conservação
Regra de ouro: SEMPRE use validação cruzada espacial para SDM.
No TAIPA: selecione "Espacial (Recomendada para SDM)". Buffer padrão: 10 km.

Métricas de Avaliação

Matriz de confusão

A base de todas as métricas. Compara o que o modelo previu com a realidade:

PREDITO pelo modelo Presente Ausente ┌──────────────┬──────────────┐ Presente │ TP │ FN │ REAL │ (acertou │ (perdeu │ │ presença) │ presença) │ ├──────────────┼──────────────┤ Ausente │ FP │ TN │ │ (falso │ (acertou │ │ alarme) │ ausência) │ └──────────────┴──────────────┘

Métricas de Avaliação

Analogia: detector de incêndio florestal

TP — Alarme toca e tinha incêndio
Salvou a floresta!
FN — NÃO tocou e tinha incêndio
PERIGOSO! Floresta queima!
FP — Alarme toca sem incêndio
Equipe vai à toa, mas sem dano
TN — Não tocou e não tinha incêndio
Tudo certo!

Métricas de Avaliação

Métricas derivadas da matriz

MétricaFórmula simplificadaPergunta que responde
Acurácia (TP + TN) / Total No geral, quantas vezes acertei?
Precisão TP / (TP + FP) Quando disse "presente", estava certo?
Sensibilidade TP / (TP + FN) Das presenças reais, quantas encontrei?
F1-Score Média harmônica Prec/Sens Equilíbrio entre precisão e sensibilidade
Para SDM, Sensibilidade é especialmente importante — queremos minimizar FN e encontrar todas as áreas de ocorrência potencial.

Métricas de Avaliação

AUC-ROC

A AUC (Area Under the ROC Curve) mede a capacidade do modelo de distinguir entre presença e ausência em todos os limiares possíveis.

Escala de interpretação da AUC: 0.5 0.6 0.7 0.8 0.9 1.0 │──────────│──────────│──────────│──────────│──────────│ Aleatório Fraco Aceitável Bom Excelente (moeda)
AUC é a métrica mais usada em publicações de SDM. Independe do limiar de corte (threshold).

Métricas de Avaliação

TSS (True Skill Statistic)

TSS = Sensibilidade + Especificidade - 1
-1.0 0.0 0.4 0.6 0.8 1.0 │──────────│──────────│──────────│──────────│──────────│ Pior que Aleatório Aceitável Bom Excelente Perfeito aleatório
  • Combina sensibilidade com especificidade (-1 a +1)
  • Não depende da prevalência — recomendado para SDM

Overfitting vs. Underfitting

O dilema

Underfitting
(muito simples)

Modelo não aprendeu o suficiente.
Aluno que não estudou.
Overfitting
(muito complexo)

Modelo decorou os dados.
Aluno que decorou a prova anterior.
O ideal: modelo que entendeu os padrões e consegue aplicar em situações novas (generalizar).

Overfitting vs. Underfitting

Como identificar

SinalUnderfittingOverfitting
Métricas no treinoBaixasMuito altas (~1.0)
Métricas no testeBaixasMuito menores que treino
Diferença treino/testePequenaGRANDE
Mapa de projeçãoUniforme demaisFragmentado, "ruidoso"
Sinal de alerta: Se a AUC no treino é 0.99 mas no teste é 0.65, o modelo está decorando os dados.

Overfitting vs. Underfitting

Como resolver

Se underfitting:
- Aumentar número de árvores
- Aumentar profundidade máxima
- Adicionar variáveis relevantes
- Verificar qualidade dos dados
Se overfitting:
- Reduzir profundidade máxima
- Aumentar amostras mín. split/folha
- Remover variáveis redundantes
- Usar validação espacial
