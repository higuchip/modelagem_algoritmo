Dados de Teste

Divisão Treino-Teste

Como dividir: 80/20

Todos os dados (100%) ┌──────────────────────────────────────┬──────────┐ │ TREINO (80%) │TESTE(20%)│ │ │ │ │ O modelo aprende com esses dados │ Avalia │ └──────────────────────────────────────┴──────────┘
Regra fundamental: O modelo NUNCA vê os dados de teste durante o treinamento!
No TAIPA: O slider "Proporção de Teste" controla essa divisão (padrão: 20%).

Divisão Treino-Teste

Estratificação

A proporção de presenças e ausências deve ser mantida em ambos os conjuntos:

200 pontos totais (100 presenças + 100 ausências) Treino (80%): 160 pontos → 80 presenças + 80 ausências ✓ Teste (20%): 40 pontos → 20 presenças + 20 ausências ✓
Estratificação garante que o treino e o teste tenham a mesma proporção de classes. Sem isso, o teste poderia ter só ausências — e a avaliação seria inútil.

Random Forest: Hiperparâmetros

O que são hiperparâmetros?

Hiperparâmetros são configurações que nós definimos antes do treinamento. O modelo não aprende esses valores sozinho — somos nós que escolhemos.

Configurações diferentes produzem modelos diferentes. Vamos ver as 4 principais e o efeito prático de cada uma na modelagem da Araucária.

Hiperparâmetros

1. Número de Análises (N. Árvores)

O modelo gera várias análises independentes e combina por votação:

50 análises
Análise 1: presente
Análise 2: ausente
Análise 3: presente

Resultado: 2×1
Instável — muda a cada rodada
500 análises
Análise 1: presente
Análise 2: presente
... (500 votam)

Resultado: 420×80
Estável e confiável

Recomendado: 500 (padrão TAIPA). Aumentar não causa overfitting, só fica mais lento.

Hiperparâmetros

2. Profundidade Máxima

Cada análise faz perguntas sequenciais sobre variáveis climáticas. A profundidade limita quantas:

Profundidade = 2
Temp. > 18°C? → Precip. > 1200mm? → FIM
Só 2 perguntas → perde nuances (underfitting)
Profundidade = 20
Temp.? → Precip.? → Sazonalidade? → Temp.mín? → ...
20 perguntas → decora ruído (overfitting)
O ideal é o meio-termo (10–30): perguntas suficientes para captar padrões climáticos reais. Padrão TAIPA: 20.

Hiperparâmetros

3. Amostras Mínimas (Split e Folha)

Controlam a prudência do modelo ao criar regras. Exemplo: dividir pontos por temperatura.

Mín. Split = 2 (pouco exigente)
Grupo com 3 pontos → divide (3 ≥ 2)
Subgrupo A: 1 ponto
Subgrupo B: 2 pontos

Regra baseada em quase nada!
Mín. Split = 5 (mais exigente)
Grupo com 3 pontos → não divide (3 < 5)
Mantém o grupo junto


Mais seguro, menos ruído

Mín. Split (padrão: 5): mínimo de pontos para dividir um grupo. Mín. Folha (padrão: 2): mínimo de pontos em cada grupo final.

Hiperparâmetros

Resumo e valores recomendados

N. Árvores500Resultado instávelSó fica mais lento

Validação Cruzada

O problema do split único

Um único split 80/20 pode dar sorte ou azar. E se os 20% de teste forem pontos "fáceis"? Ou "difíceis"?

Analogia: Avaliar um aluno com uma única prova vs. várias provas ao longo do semestre.

  • Uma prova: nota pode não refletir o conhecimento real
  • Várias provas: média das notas é mais confiável
Validação cruzada é como aplicar várias provas — cada uma com perguntas diferentes.

Validação Cruzada

K-Fold Cross-Validation

Dividimos os dados em K partes iguais (folds). Cada parte é usada como teste exatamente uma vez:

Fold 1: [TESTE] [treino] [treino] [treino] [treino] Fold 2: [treino] [TESTE] [treino] [treino] [treino] Fold 3: [treino] [treino] [TESTE] [treino] [treino] Fold 4: [treino] [treino] [treino] [TESTE] [treino] Fold 5: [treino] [treino] [treino] [treino] [TESTE] Resultado final = média das 5 avaliações

Com K=5, o modelo é treinado e avaliado 5 vezes, cada vez com dados de teste diferentes.

Validação Cruzada

Vantagens do K-Fold

  • Todos os dados são usados para treino E para teste
  • Resultado mais robusto (média de K avaliações em vez de 1)
  • Detecta melhor se o modelo está generalizando
  • Mostra a variabilidade do desempenho (desvio padrão entre folds)
K = 5 é o valor mais comum. Significa 5 rodadas de treino/teste com partes diferentes.
No TAIPA: A validação cruzada vem ativada por padrão. O slider "Número de Folds" permite ajustar (padrão: 5).

Validação Cruzada Espacial

O problema da autocorrelação espacial

Autocorrelação espacial: pontos próximos no espaço tendem a ter condições ambientais similares.

Se um ponto de treino está muito perto de um ponto de teste, o modelo pode "colar" da vizinhança em vez de aprender relações climáticas reais.

Ponto de treino (A) ← 5 km → Ponto de teste (B) A e B têm clima quase idêntico! O modelo acerta B sem ter aprendido nada... apenas porque A estava ao lado.

Validação Cruzada Espacial

Analogia: prova com cola vs. prova real

CV padrão em SDM

Como fazer prova sentado ao lado de quem já sabe as respostas.

Nota alta, mas não aprendeu nada!
CV espacial

Como fazer prova em sala separada, sem consulta.

Nota mais realista — mostra o que realmente aprendeu.

Validação Cruzada Espacial

Como funciona na prática

Passo 1: Agrupar pontos por localização geográfica (KMeans) ┌───────────┐ ┌───────────┐ ┌───────────┐ │ Grupo 1 │ │ Grupo 2 │ │ Grupo 3 │ │ (Norte) │ │ (Centro) │ │ (Sul) │ │ * * * │ │ * * * │ │ * * * │ └───────────┘ └───────────┘ └───────────┘ ↕ buffer ↕ buffer Passo 2: Cada grupo vira conjunto de teste Passo 3: Buffer remove pontos de treino próximos à fronteira

Resultado: métricas refletem a capacidade real de prever em áreas novas.

Validação Cruzada Espacial

Por que é ESSENCIAL para SDM

Validação padrão pode dar AUC de 0.95 quando a capacidade real é 0.75!
  • Dados ecológicos sempre têm autocorrelação espacial
  • O objetivo do SDM é prever em novas áreas
  • Métricas infladas levam a decisões erradas de conservação
Regra de ouro: SEMPRE use validação cruzada espacial para SDM.
No TAIPA: selecione "Espacial (Recomendada para SDM)". Buffer padrão: 10 km.

Métricas de Avaliação

Matriz de confusão

A base de todas as métricas. Compara o que o modelo previu com a realidade:

PREDITO pelo modelo Presente Ausente ┌──────────────┬──────────────┐ Presente │ VP │ FN │ REAL │ (acertou │ (perdeu │ │ presença) │ presença) │ ├──────────────┼──────────────┤ Ausente │ FP │ VN │ │ (falso │ (acertou │ │ alarme) │ ausência) │ └──────────────┴──────────────┘

Na prática: expedição de campo

Equipe usa o modelo para decidir quais fragmentos visitar em busca de Araucária:

VP — Acerto de presença
Modelo disse "tem" → equipe confirma.
📋 Entra no plano de conservação.
FN — Presença perdida
Modelo disse "não tem" → área desmatada.
🚨 População perdida.
FP — Alarme falso
Modelo disse "tem" → não encontra.
⚠️ Gastou diária, sem dano ambiental.
VN — Acerto de ausência
Modelo disse "não tem" → não tinha.
✅ Tudo certo.
Erro mais grave: FN — perder uma população é irreversível. Um FP só custa uma viagem.

Métricas de Avaliação

Métricas derivadas da matriz

MétricaFórmulaO que medeExemplo com Araucária
Acurácia (VP + VN) / Total % de acertos no geral De 100 fragmentos, em quantos o modelo acertou?
Precisão VP / (VP + FP) Confiabilidade do "sim" Dos que o modelo disse "tem Araucária", quantos realmente tinham?
Sensibilidade VP / (VP + FN) Capacidade de encontrar Das áreas onde a Araucária existe, quantas o modelo encontrou?
F1-Score Média harmônica Equilíbrio Prec/Sens Nota geral que penaliza se uma das duas for muito baixa

Métricas de Avaliação

Calculando passo a passo

1. Acurácia = (VP + VN) / Total = (35 + 40) / 100 = 75%

De 100 fragmentos, o modelo acertou 75. Parece bom, mas será que conta toda a história?

2. Precisão = VP / (VP + FP) = 35 / (35 + 15) = 35/50 = 70%

Dos 50 fragmentos que o modelo disse "tem Araucária", 35 realmente tinham. 15 foram alarmes falsos.

3. Sensibilidade = VP / (VP + FN) = 35 / (35 + 10) = 35/45 = 77,8%

Dos 45 fragmentos onde a Araucária existia, o modelo encontrou 35. Mas perdeu 10!

Reflexão: A acurácia de 75% esconde que o modelo perdeu 10 fragmentos com Araucária. Para conservação, esses 10 "perdidos" podem ser populações críticas!

Precisão vs. Sensibilidade: cenário real

SC tem verba para 50 fragmentos. Modelo indicou 80 como "com Araucária" — 60 realmente têm.

A: Priorizar Precisão
Visita os 50 mais prováveis
→ 45 confirmados, 5 falsos
→ 90% aproveitamento
→ 15 populações de fora
Bom: verba curta
B: Priorizar Sensibilidade
Visita todos os 80
→ encontra as 60 reais
→ 20 viagens "à toa"
→ nenhuma perdida
Bom: espécie ameaçada
Araucária (ameaçada): Estratégia B é mais segura — 20 diárias extras valem menos que perder populações.

Métricas de Avaliação

Qual métrica priorizar? Depende do objetivo!

SituaçãoPriorizarPor quê
Criar unidade de conservação (verba limitada) Precisão Cada hectare custa caro — precisa acertar
Buscar populações para coleta de sementes Sensibilidade Não pode perder nenhuma população
Plano de manejo / publicação científica F1-Score Equilíbrio entre não perder e não desperdiçar

Não existe métrica "melhor". A escolha depende da pergunta ecológica e das consequências dos erros.

Métricas de Avaliação

AUC e TSS

Duas métricas muito usadas em SDM que resumem a qualidade geral do modelo:

ValorAUC (0 a 1)TSS (-1 a +1)
Ruim≤ 0,6 — não aprendeu nada≤ 0 — pior que chutar

AUC = o quanto o modelo consegue separar presenças de ausências.
TSS = combina sensibilidade e especificidade. Funciona bem mesmo quando há muito mais ausências do que presenças nos dados (o que é comum em SDM).

Overfitting vs. Underfitting

O dilema

Underfitting
(muito simples)

Modelo não aprendeu o suficiente.
Aluno que não estudou.
Overfitting
(muito complexo)

Modelo decorou os dados.
Aluno que decorou a prova anterior.
O ideal: modelo que entendeu os padrões e consegue aplicar em situações novas (generalizar).

Overfitting vs. Underfitting

Como identificar

SinalUnderfittingOverfitting
Métricas no treinoBaixasMuito altas (~1.0)
Métricas no testeBaixasMuito menores que treino
Diferença treino/testePequenaGRANDE
Mapa de projeçãoUniforme demaisFragmentado, "ruidoso"
Sinal de alerta: Se a AUC no treino é 0.99 mas no teste é 0.65, o modelo está decorando os dados.

Overfitting vs. Underfitting

Como resolver

Se underfitting:
- Aumentar número de árvores
- Aumentar profundidade máxima
- Adicionar variáveis relevantes
- Verificar qualidade dos dados
Se overfitting:
- Reduzir profundidade máxima
- Aumentar amostras mín. split/folha
- Remover variáveis redundantes
- Usar validação espacial

Checklist: "Meu modelo está pronto?"

Antes de gerar mapas ou escrever relatório, passe por estas 4 perguntas:

1. AUC da CV espacial está acima de 0.7?
Sim → aceitável para relatório. Não → revise variáveis ou hiperparâmetros.
2. A variável mais importante faz sentido ecológico?
Ex: BIO6 para Araucária — geada é limitante no Sul. Se for estranha, pode ser ruído.
3. A diferença treino vs. teste é grande?
AUC treino 0.99 vs teste 0.65 → overfitting! Reduza profundidade ou use CV espacial.
4. A sensibilidade está adequada para o objetivo?
Conservação ameaçada → sensibilidade > 0.7. Consultoria com verba curta → precisão alta.
